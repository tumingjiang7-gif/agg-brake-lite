# AGG Brake Lite Support Policy

## Current Support Level

This project is intentionally small and low-touch.

Current support includes:

- bug fixes for the current release window
- clarifications through public documentation
- structured pilot feedback through a shared template

Current support does **not** include:

- private consulting
- custom integrations
- production guarantees
- service-level commitments
- custom rule development on request

## Best Way To Send Useful Feedback

Use:

- [FAQ.md](/mnt/e/agpt/automaton_unpacked/automaton-main/sdk/agg_brake_lite/FAQ.md)
- [FEEDBACK_TEMPLATE.md](/mnt/e/agpt/automaton_unpacked/automaton-main/sdk/agg_brake_lite/FEEDBACK_TEMPLATE.md)

The most useful feedback includes:

- where the check was added
- what amount limit was used
- whether the warning was right or wrong
- whether an anonymous export was attached

## Response Boundary

This project is not run as a high-touch service.

If feedback arrives, the default path is:

1. review whether it matches the narrow scope
2. decide whether it helps the next release
3. update docs or code only if it strengthens the core payment-brake use case

## Scope Boundary

This project stays focused on:

- local warning for AI checkout/payment automation

Requests outside that slice may be documented, but they do not automatically become roadmap items.
