# AGG Brake Lite Validation Playbook

## Goal

Use the smallest possible launch to answer one question:

- does this narrow local payment brake deserve more time?

## Validation Window

Use a 14-day first window.

### Day 0

- publish repository
- set one real payment link
- publish one small announcement set

### Day 3

Check only:

- did anyone pay
- did anyone install and respond
- did anyone ask a concrete integration question

### Day 7

Check whether any signal became real:

- stranger paid
- stranger exported anonymous logs
- stranger described a real integration

### Day 14

Make the first hard decision:

- continue
- archive

Do not create a vague "maybe later" state.

## Real Signals

Only these count:

1. stranger paid
2. stranger exported anonymous logs
3. stranger shared a real integration result

## Non-Signals

These do not justify more product work by themselves:

- stars
- downloads
- compliments
- curiosity
- framework requests without real usage

## Continue Rule

Continue only if at least one real signal exists by day 14.

If the signal is weak, continue narrowly:

- keep scope narrow
- do not expand beyond checkout/payment
- only build the next missing step

## Archive Rule

Archive if day 14 ends with:

- no stranger payment
- no anonymous export
- no real integration result

The point is to protect time, not to defend sunk effort.
